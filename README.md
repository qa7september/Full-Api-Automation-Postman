# 🧪 Testes Automatizados de API com Postman

Projeto de automação de testes de API com foco na validação de endpoints REST, status codes e cenários com dados válidos e inválidos.

---

🚀 Tecnologias utilizadas

- Postman
- JavaScript (scripts de teste)
- REST API

---

🎯 Objetivo do projeto

Garantir a qualidade e confiabilidade da API através de testes automatizados, validando comportamentos esperados e identificando possíveis falhas.

---

🔍 Escopo dos testes

- Testes nos métodos HTTP: GET, POST, PUT, DELETE
- Validação de status codes
- Testes com dados válidos (positivos)
- Testes com dados inválidos (negativos)
- Validação de respostas da API (body, estrutura e dados)

---

🧠 Estratégia de testes

- Foco em endpoints críticos da aplicação
- Cobertura de cenários positivos e negativos
- Validação de comportamento inesperado
- Organização dos testes para fácil manutenção e reutilização

---

🗂️ Estrutura do projeto

- Collections organizadas por endpoints
- Scripts automatizados para validação
- Requisições estruturadas para facilitar execução

---

📈 Resultados alcançados

- Identificação de falhas e inconsistências na API
- Validação completa dos principais endpoints
- Melhoria na confiabilidade e estabilidade do sistema

---

## ▶️ Como executar o projeto

1. Importar a collection no Postman
2. Configurar o environment (se necessário)
3. Executar as requisições ou rodar a collection completa
